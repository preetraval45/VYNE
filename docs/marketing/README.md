# VYNE marketing pack

Six artifacts you'll use in the 48 hours around launch.

| File | When to use it |
|------|----------------|
| [launch-day-script.md](launch-day-script.md) | Hour-by-hour playbook from T-2d → T+7d |
| [press-release.md](press-release.md) | Hand to journalists who say yes to a piece |
| [email-to-press.md](email-to-press.md) | Cold-outreach template + target list |
| [launch-tweet.md](launch-tweet.md) | 12-tweet thread, schedule for 8am ET |
| [product-hunt.md](product-hunt.md) | PH listing, taglines, hunter outreach |
| [hacker-news.md](hacker-news.md) | Show HN title + first comment + reply prep |

Sequence: write/customize the launch-tweet.md and product-hunt.md first (they're the two highest-leverage assets). Use launch-day-script.md as the live timeline. Press release + press email are for the second wave once organic momentum proves the product.

## Customize before sending

Search for these tokens and replace:

- `[LAUNCH DATE]`, `[CITY, DATE]`, `[Name]`, `[SPECIFIC PIECE — make this real or skip]`
- `[TEAM SIZE]`, `[your phone]`, `[your X handle]`, `[Customize: yes / no / not yet / DM me]`
- `vyne.dev` / `noreply@yourdomain.com` if you have a custom apex domain
- `Preet Raval` / pronouns / specific founder voice elsewhere

## Don't skip

- The pre-warm DM list (T-1d in the launch-day script). Cold launches die.
- The Stripe live-mode dry run. A broken pricing page on launch day is a non-recoverable miss.
- The first PH comment within 60 seconds of going live. Late = invisible.
- Sentry alert routing. P0 errors during the launch hour need to ping your phone.
